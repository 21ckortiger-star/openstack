goonggeum 홈페이지 프로젝트

발표자료 (PDF 변환용)

1. 프로젝트 소개

  • 개인 블로그형 미니 홈페이지 프로젝트
  • 우주 테마 UI + 로그인 기반 접근 제어 + DB 연동
  • 정적 페이지를 동적 웹 서비스로 확장

2. 문제 정의 및 목표

문제

  • 정적 페이지는 사용자 상태 기반 기능 제어가 어려움
  • 프로젝트 상세 접근 권한 제어 부재
  • 회원가입/로그인 기능 부재

목표

  • 회원가입/로그인/로그아웃 구현
  • 로그인 사용자만 프로젝트 상세 접근
  • 세련된 홈페이지 UI/UX 구현

3. 주요 기능

  • 메인 홈페이지 구성 (About / Projects / Contact)
  • 회원가입/로그인/로그아웃 + 세션 유지
  • BCrypt 비밀번호 암호화
  • 로그인 기반 프로젝트 접근 제어
  • 비로그인 상태 카드 잠금(🔒) 표시

4. 시스템 구조 (아키텍처)

  • Frontend: Thymeleaf + HTML/CSS/JS
  • Backend: Spring Boot (Controller-Service-Repository)
  • Database: MySQL

처리 흐름: 로그인 요청 → Controller → Service → Repository(DB) → Session 저장 → 프로젝트 접근 시 세션 검증

5. 데이터베이스 구조 (ERD)

member

  • username (PK)
  • password (BCrypt)
  • name

course (샘플)

  • id (PK), title, instructor_name, description

6. 시연 및 결과

  • 메인 접속 후 잠금 UI 확인
  • 회원가입 / 로그인 수행
  • 로그인 후 프로젝트 상세 진입 성공
  • 로그아웃 후 접근 제한 재확인

결과: 인증 기능, 접근 제어, UI 개선 목표 달성

7. 한계점 및 개선 계획

한계점

  • 단일 사용자 권한 구조
  • 정적 콘텐츠 비중 높음
  • 보안 고도화 기능 미적용

개선 계획

  • RBAC(관리자/사용자) 도입
  • 게시글/프로젝트 CRUD 추가
  • 댓글/좋아요 등 블로그 기능 확장
  • 배포 자동화 및 모니터링
