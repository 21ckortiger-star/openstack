# goonggeum 홈페이지 프로젝트 발표자료

> 형식: PDF (10페이지 이내 권장)

---

## 1. 프로젝트 소개
- 프로젝트명: **goonggeum Homepage Project**
- 프로젝트 성격: 개인 블로그형 미니 홈페이지
- 핵심 키워드: **우주 테마 UI + 로그인 기반 접근 제어 + DB 연동**
- 개발 목적: 정적 페이지를 동적 웹 서비스로 확장하여 인증/데이터 연동 경험 확보

---

## 2. 문제 정의 및 목표
### 문제 정의
- 기존 정적 홈페이지는 사용자 상태(로그인 여부)에 따라 기능을 제어하기 어려움
- 프로젝트 상세 페이지 접근 권한 제어 부재
- 사용자 계정 관리(회원가입/로그인) 기능 부재

### 목표
- 회원가입/로그인/로그아웃 기능 구현
- 로그인 사용자만 프로젝트 상세 페이지 접근 가능하도록 제어
- 개인 홈페이지 감성에 맞는 세련된 UI/UX 구현

---

## 3. 주요 기능
1. **메인 홈페이지 구성**
   - About / Projects / Contact 섹션
   - 우주 이미지 기반 프로젝트 카드 3종

2. **인증 기능**
   - 회원가입, 로그인, 로그아웃
   - 세션 기반 로그인 상태 유지
   - BCrypt 비밀번호 암호화 저장

3. **접근 제어 기능**
   - `/project/1~3` 로그인 사용자만 접근
   - 비로그인 시 로그인 페이지로 리다이렉트
   - 비로그인 상태에서 카드 잠금(🔒) UI 표시

4. **UI 개선**
   - 로그인 페이지 중앙 정렬 + 카드형 디자인
   - 로그인 버튼/입력창 hover 힌트(데모용)

---

## 4. 시스템 구조 (아키텍처)
### 구성
- **Frontend**: Thymeleaf 템플릿 + HTML/CSS/JS
- **Backend**: Spring Boot (Controller / Service / Repository)
- **Database**: MySQL

### 처리 흐름
1) 사용자가 `/login` 요청
2) `AuthController` → `AuthService` 인증 처리
3) `AuthRepository`가 DB(`member`) 조회
4) 성공 시 Session(`LOGIN_USER`, `LOGIN_NAME`) 저장
5) `/project/*` 접근 시 `ProjectController`에서 세션 검증

---

## 5. 데이터베이스 구조 (ERD)
### member 테이블
- `username` (PK)
- `password` (BCrypt 해시)
- `name`

### course 테이블 (샘플 콘텐츠)
- `id` (PK)
- `title`
- `instructor_name`
- `description`

### ERD 요약
- `member`: 인증/사용자 정보 관리
- `course`: 프로젝트 내 샘플 데이터 표현

---

## 6. 시연 및 결과
### 시연 시나리오
1. 메인 페이지 접속 (`/`)
2. 비로그인 상태에서 프로젝트 카드 잠금 표시 확인
3. 회원가입 (`/signup`) 진행
4. 로그인 (`/login`) 성공
5. 메인에서 사용자명 표시 확인
6. 프로젝트 카드 클릭 후 상세 페이지 진입 성공
7. 로그아웃 후 접근 제한 재확인

### 결과
- 로그인/로그아웃/회원가입 정상 동작
- 로그인 여부 기반 접근 제어 정상 동작
- 목표한 UI 개선 및 사용자 흐름 구현 완료

## 7. 한계점 및 개선 계획
### 한계점
- 권한 체계가 단순(일반 사용자 단일 역할)
- 프로젝트 콘텐츠가 정적/하드코딩 중심

3. 댓글/좋아요 등 **블로그형 기능** 추가
4. 마이페이지(프로필 수정, 비밀번호 변경) 구현
5. 배포 자동화 및 모니터링 도입
